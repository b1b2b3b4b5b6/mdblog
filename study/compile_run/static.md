# static
* 定义时, 被修饰的符号不参与链接时的决议(除了成员函数), 但还是影响内存的排布
* 要使static变量被直接或间接使用，不然编译会忽略此static变量


### C++
在C语言的基础上，还添加了以下用法
| 修饰对象 |                作用                | 初始化时机|
| -------- | ---------------------------------- |---- |
| 全局变量 | 变量作用域为此源文件                         | main之前 |
| 局部变量 | 局部变量转移到.data或.bss中存储，生命周期全程 |第一次调用时 |
| 函数     | 函数作用域为此源文件                         | /|
| 成员函数内的局部变量 |局部变量转移到.data或.bss中存储，生命周期全程               |第一次调用时 |
| 成员函数 | 函数调用时可以不需要this指针，通过类名调用                | /|
| 成员变量 | 成员转移到.data或.bss中存储，生命周期全程 | main之前|
> 如果局部变量同时被const修饰, 且初始值可确定, 则会转移到.rodata中存储
> 类的static成员变量须在全局定义(是否初始化可选, 不可用static修饰), 类内仅是声明
> 类的static成员函数可在类内定义, 为弱符号
> 类的static成员函数或成员变量在全局定义时, 可定义为弱符号

int OBJ::a = 10;
